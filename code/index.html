Code

I like to program computers, and I've done it as a hobby, for work, and for class. I program mostly in C/C++, Python, and Ruby, although I know a few other languages, too. Below are the projects or whatever that I feel are worth sharing, either because they might be useful or they demonstrate something cool. Or at least something that I think is cool. Most of them weren't designed with sharing in mind, so I'll only be adding things as they become suitable for distribution.

Personal

  • LDAP email widget

    The LDAP email widget is, well, just what it says. This was my first ever AJAX program and I borrowed some ideas from Google Suggest (e.g., the timeout adjust feature). It acts like an LDAP-enabled email client, querying the University of Chicago's LDAP server for email addresses and/or names matching what you enter into the boxes. They are attached using ids, so you can have as many on a single page as you'd like. The Javascript is well-structured but would look funny to someone familiar with Javascript. I definitely wouldn't write it like this now, which goes to show what two years of JS development experience will get you, I guess.

Academic

  • EAI

    EAI (Extensible Authorship Identification) is a modular system for determining to which of two authors a given text is most similar. It is written in Python and Perl, although I plan to reimplement it purely in Python so as to make certain speed improvements. This was my final project for CMSC 35100, Natural Language Processing. I hope to continue working on it so as to make it function more reliably.

  • suffixes

    suffixes is a C++ program which tries to compute root/suffix boundaries within a word given a training corpus. By default it uses the supplied KJV Bible as training data, but it can use any file so long as the file has punctuation and capitalization removed. If the corpus isn't prepared in this way it will treat punctuation as letters and not be able to differentiate between 'a' and 'A', for example. It processes one string at a time, but can read from standard input so that it can parse an external list of words. It is language agnostic, but assumes ASCII input. If the input contains tab-delimited data corresponding to the correct parse of a word, suffixes will also compute its precision, recall, and F-score.

  • viterbi

    viterbi is a C++ program which uses a Viterbi-style parsing algorithm to try and turn a string of letters into words. That is, given an input of "therentisdue" its output might be something like "-the-rent-is-due," presuming it is actually able to predict this. By default it, too, uses the KJV Bible as training data (which can produce some amusing results, e.g., it assigns a high probability to the word "sin"), but a different corpus can be specified via the command line.

Professional

  • brutelogin

    brutelogin is a custom replacement of Apple's Login.app for Mac OS X written in C, Objective-C, and bourne shell script. It is in use on approximately 30 computers throughout the University of Chicago's housing system. For legal reasons it is unavailable for download, but the above link contains a brief overview of how it works and some screenshots. At one point I was the primary developer and maintainer for this application, preceeded by Alex Hioreanu and followed by Jacob Lilly. My work enabled it to be put into production use on the Rescom lab machines.

  • Chopin

    Chopin is an "enterprise management system" LAMP project used by RH&C, OUSH, and Rescom at the University of Chicago. The students know it mostly because it enables them to file housing-and-computer-related workorders, but it does a lot more, including keeping track of budgetary information, housing assignments, and so forth. Basically if it's housing-related it's done in Chopin. The above link will take you to the Chopin homepage, which will give you a vague idea of the more simple aspects of what it does even if you don't have access.
